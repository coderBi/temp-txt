有些时候，git已经commit了，甚至是已经push了。这个时候如果发现commit的信息写错了或者是由于某种原因需要对这个信息进行修改，git中提供了修改上一次commit的方法。通过在git shell中使用指令git commit --amend可以修改上一次提交的描述信息。但是要注意这个修改只能用于上一次提交，也就是说如果一个提交如果不是最新的提交，那么它的描述信息没有办法进行修改了。
git commit --amend 不会创建一个新的commit对这次的更改进行描述，而是在上一个的commit点直接进行修改。所以通过git commit --amend指令修改之后，git的最后一个commit还是没有变，这也就意味着一个最新的commit的信息是可以进行连续的修改的。
